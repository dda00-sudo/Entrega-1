# Historia y Objetivo de Git

## ¿Quién lo creó y por qué?
**Git** fue creado en 2005 por **Linus Torvalds**, el mismo creador del kernel de Linux.

La razón principal fue la necesidad de una herramienta de control de versiones **distribuida** que pudiera manejar de forma eficiente el desarrollo del kernel de Linux, que es un proyecto enorme con miles de colaboradores.

## Objetivo y Uso Actual
El objetivo fundamental de Git es llevar un registro completo y seguro de **todos los cambios** realizados en el código fuente o en un conjunto de archivos a lo largo del tiempo.

Hoy en día, Git es el sistema de control de versiones más popular del mundo y se utiliza para:
1.  **Colaboración:** Permite que múltiples desarrolladores trabajen en el mismo proyecto simultáneamente sin interferir entre sí.
2.  **Rastreo:** Permite volver a cualquier versión anterior del proyecto si surge un problema (revertir cambios).
3.  **Ramificación (Branching):** Permite aislar nuevas características o correcciones de errores en ramas separadas, manteniendo la rama principal estable.
